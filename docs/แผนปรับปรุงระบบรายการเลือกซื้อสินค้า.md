# แผนการพัฒนาปรับปรุงระบบรายการเลือกซื้อสินค้า

> โปรเจกต์: **Fun Market / ตลาดอาหาร 5 หมู่ ป.3**  
> Repository: `shanwinder/fun_market`  
> เป้าหมาย: เพิ่มหน้า **รายการสินค้าให้เลือกซื้อ** สำหรับนักเรียน โดยไม่กระทบระบบ QR Code เดิม และไม่เฉลยหมู่อาหาร/คุณค่าทางอาหารระหว่างกิจกรรม

---

## 1. ภาพรวมปัญหาและความต้องการ

ระบบปัจจุบันของ `fun_market` มี flow หลักของนักเรียนคือ

```text
เข้ากลุ่ม → สแกน QR Code → หน้าสินค้า → เพิ่มลงตะกร้า → ดูตะกร้า → ยืนยันซื้อ
```

ระบบนี้เหมาะกับกิจกรรมแบบเดินฐานหรือวางบัตรสินค้าให้สแกน แต่ในการใช้งานจริงอาจมีปัญหา เช่น

- กล้องมือถือบางเครื่องเปิดไม่ได้
- นักเรียนบางกลุ่มไม่มีอุปกรณ์พร้อมสแกน
- อินเทอร์เน็ตหรือ permission กล้องมีปัญหา
- ครูต้องการให้เด็กเลือกสินค้าจากหน้าจอโดยตรง
- การสแกนทีละสินค้าอาจใช้เวลามากในชั้นเรียน ป.3

ดังนั้นควรเพิ่มอีก flow หนึ่งคือ

```text
เข้ากลุ่ม → ดูรายการสินค้า → เลือกสินค้า → เพิ่มลงตะกร้า → ยืนยันซื้อ
```

โดยระบบ QR Code เดิมยังคงอยู่เหมือนเดิม

## 2. หลักการสำคัญด้านการเรียนรู้

ระบบรายการเลือกซื้อสินค้าต้องยังยึดหลักเดิมของโปรเจกต์ คือ

> **ไม่เฉลยให้นักเรียนเห็นว่าอาหารแต่ละชนิดอยู่หมู่ใด หรือมีคุณค่าทางอาหารอย่างไร ระหว่างทำกิจกรรม**

### 2.1 ข้อมูลที่แสดงให้นักเรียนเห็นได้

ในหน้า `student/products.php` แสดงได้เฉพาะข้อมูลกลาง ๆ เช่น

- รูปสินค้า
- ชื่อสินค้า
- รายละเอียดสั้น ๆ แบบไม่เฉลย เช่น `ข้าวสวย 1 จาน`, `นมจืด 1 กล่อง`
- ราคา
- จำนวนที่เลือก
- ปุ่มเพิ่มลงตะกร้า
- เงินคงเหลือของกลุ่ม
- จำนวนรายการในตะกร้า

### 2.2 ข้อมูลที่ห้ามแสดงในหน้ารายการสินค้า

ห้ามแสดงข้อมูลเหล่านี้ในฝั่งนักเรียน

- หมู่อาหาร
- คุณค่าทางอาหาร
- คำว่า โปรตีน / คาร์โบไฮเดรต / วิตามิน / เกลือแร่ / ไขมัน
- สีแยกหมวดอาหาร
- ไอคอนที่ใบ้หมวดอาหาร
- ข้อความแนะนำว่าควรซื้ออะไร
- คะแนนความเหมาะสม
- `teacher_note`

## 3. สถานะระบบปัจจุบันที่เกี่ยวข้อง

### 3.1 ฐานข้อมูลรองรับอยู่แล้ว

ตาราง `products` มีข้อมูลที่เพียงพอสำหรับทำหน้ารายการสินค้า ได้แก่

```text
id
activity_id
product_name
description
price
image_path
qr_token
stock_qty
is_active
sort_order
teacher_note
created_at
```

ตาราง `carts` และ `cart_items` มีไว้รองรับตะกร้าของกลุ่มอยู่แล้ว

```text
carts
- id
- activity_id
- group_id
- status
- created_at
- updated_at

cart_items
- id
- cart_id
- product_id
- quantity
- created_at
```

ดังนั้นการเพิ่มหน้ารายการสินค้า **ไม่จำเป็นต้องเพิ่มตารางใหม่** ในระยะแรก

---

### 3.2 ระบบ helper รองรับตะกร้าอยู่แล้ว

ไฟล์ที่เกี่ยวข้อง

```text
includes/helpers.php
```

มี helper สำคัญอยู่แล้ว เช่น

```text
get_or_create_open_cart()
cart_items()
cart_total()
product_image_url()
sort_products_by_thai_name()
require_student_group()
```

ดังนั้นหน้า `student/products.php` สามารถใช้ helper เดิมได้ทันที

---

### 3.3 ระบบเพิ่มสินค้าลงตะกร้ามีอยู่แล้ว

ไฟล์ที่เกี่ยวข้อง

```text
actions/add_to_cart.php
```

action นี้รับข้อมูล

```text
product_id
quantity
csrf_token
```

แล้วตรวจสอบว่า

- นักเรียนเลือกกลุ่มแล้ว
- สินค้าอยู่ในกิจกรรมเดียวกับกลุ่ม
- สินค้ายังเปิดขาย
- กิจกรรมยัง active

จากนั้นเพิ่มสินค้าลงตะกร้าเดิม ถ้าสินค้าซ้ำจะบวกจำนวนเพิ่ม

ดังนั้นหน้า `student/products.php` สามารถใช้ action นี้ต่อได้เลย

---

### 3.4 ระบบ checkout ปลอดภัยอยู่แล้ว

ไฟล์ที่เกี่ยวข้อง

```text
actions/checkout.php
```

ระบบ checkout ดึงราคาจากฐานข้อมูลฝั่ง server อีกครั้ง ไม่เชื่อราคาจากหน้าเว็บ และใช้ transaction กับ `FOR UPDATE` เพื่อลดปัญหากดซื้อพร้อมกันแล้วเงินผิดพลาด

ดังนั้นหน้ารายการสินค้าใหม่ต้องส่งเฉพาะ

```text
product_id
quantity
```

และ **ห้ามส่งราคา** ไปจาก form

## 4. แนวทางพัฒนาที่แนะนำ

แนะนำให้พัฒนาเป็น 2 ระยะ

| ระยะ | แนวทาง | เหมาะกับ |
|---|---|---|
| Phase 1 | เพิ่มหน้าเลือกสินค้าแบบกดเพิ่มทีละรายการ | ทำเร็ว เสี่ยงน้อย ใช้ระบบเดิม |
| Phase 2 | เพิ่มระบบเลือกหลายรายการแล้วกดเพิ่มครั้งเดียว | เหมาะกับ ป.3 ใช้งานเร็วขึ้น |

ควรทำ Phase 1 ก่อน เพื่อให้ระบบใช้งานได้เร็วและทดสอบง่าย จากนั้นค่อยพัฒนา Phase 2 ถ้าต้องการ UX ที่ดีกว่าสำหรับเด็กเล็ก

## 6. เพิ่มไฟล์ `student/products.php`

สร้างไฟล์ใหม่

```text
student/products.php
```

หน้าที่ของไฟล์นี้

1. ตรวจว่านักเรียนเลือกกลุ่มแล้ว
2. ตรวจว่ากิจกรรมยัง active
3. ดึงตะกร้าปัจจุบันของกลุ่ม
4. ดึงสินค้าที่เปิดขายในกิจกรรมนั้น
5. เรียงชื่อสินค้าให้เหมาะกับภาษาไทย
6. แสดงสินค้าเป็น card/grid
7. ให้เลือกจำนวนและเพิ่มลงตะกร้า
8. มีปุ่มไปตะกร้า / กลับหน้ากลุ่ม / สแกน QR

ตัวอย่างโค้ดตั้งต้น

```php
<?php

// เปิด strict mode เพื่อช่วยลดข้อผิดพลาดของชนิดข้อมูล
declare(strict_types=1);

// โหลด helper และ CSRF
require_once __DIR__ . '/../includes/helpers.php';
require_once __DIR__ . '/../includes/csrf.php';

// ตรวจว่านักเรียนเลือกกลุ่มแล้วหรือยัง
$group = require_student_group();

// ถ้ากิจกรรมปิดแล้ว หรือกลุ่มถูกปิด ให้กลับไปหน้าเลือกกลุ่ม
if ($group['activity_status'] !== 'active' || !$group['is_active']) {
    unset($_SESSION['student_group_id']);
    flash('warning', 'กิจกรรมนี้ยังไม่เปิดหรือถูกปิดแล้ว');
    redirect('student/join.php');
}

// ดึงตะกร้าปัจจุบัน เพื่อแสดงจำนวนรายการในตะกร้า
$cart = get_or_create_open_cart((int) $group['activity_id'], (int) $group['id']);
$items = cart_items((int) $cart['id']);
$cartItemCount = count($items);

// ดึงรายการสินค้าที่เปิดขายในกิจกรรมนี้
$stmt = db()->prepare(
    'SELECT id, product_name, description, price, image_path, sort_order
     FROM products
     WHERE activity_id = ?
       AND is_active = 1
     ORDER BY product_name ASC, sort_order ASC, id ASC'
);
$stmt->execute([(int) $group['activity_id']]);
$products = $stmt->fetchAll();

// เรียงชื่อภาษาไทยให้เป็นธรรมชาติมากขึ้น ถ้า server รองรับ intl extension
sort_products_by_thai_name($products);

$pageTitle = 'เลือกซื้อสินค้า';
$bodyClass = 'student-shell';

require_once __DIR__ . '/../includes/header.php';
?>

<div class="d-flex flex-wrap justify-content-between align-items-center gap-2 mb-4">
    <div>
        <div class="fm-overline mb-1">รายการสินค้า</div>
        <h1 class="page-title mb-0">เลือกซื้อสินค้า</h1>
        <p class="text-muted mb-0">
            <?= h($group['group_name']) ?> · เงินคงเหลือ <?= money($group['current_balance']) ?>
        </p>
    </div>

    <a class="btn btn-outline-primary btn-lg fm-btn-icon" href="<?= h(url('student/cart.php')) ?>">
        <i data-lucide="shopping-cart"></i>
        ตะกร้า
        <span class="badge text-bg-primary ms-1"><?= h((string) $cartItemCount) ?></span>
    </a>
</div>

<?php if (!$products): ?>
    <div class="fm-panel p-5 text-center">
        <i data-lucide="package-x" class="text-primary mb-3" style="width:56px;height:56px"></i>
        <p class="fs-4 text-muted mb-3">ยังไม่มีสินค้าเปิดขาย</p>
        <a class="btn btn-outline-secondary btn-lg fm-btn-icon justify-content-center" href="<?= h(url('student/home.php')) ?>">
            <i data-lucide="arrow-left"></i>
            กลับหน้ากลุ่ม
        </a>
    </div>
<?php else: ?>
    <div class="row g-3 fm-stagger">
        <?php foreach ($products as $product): ?>
            <div class="col-12 col-md-6 col-xl-4">
                <div class="fm-product-card h-100">
                    <img
                        class="fm-product-card-image"
                        src="<?= h(product_image_url($product['image_path'])) ?>"
                        alt="<?= h($product['product_name']) ?>"
                        loading="lazy"
                        decoding="async"
                    >

                    <div class="fm-product-card-body p-4">
                        <h2 class="h4 fw-bold mb-2"><?= h($product['product_name']) ?></h2>

                        <?php if (!empty($product['description'])): ?>
                            <p class="text-muted mb-3"><?= h($product['description']) ?></p>
                        <?php endif; ?>

                        <div class="fm-price-badge mb-3">
                            <i data-lucide="coins"></i>
                            <?= money($product['price']) ?>
                        </div>

                        <form method="post" action="<?= h(url('actions/add_to_cart.php')) ?>" class="fm-product-card-actions">
                            <?= csrf_field() ?>

                            <!-- ส่งเฉพาะ product_id และ quantity เท่านั้น ไม่ส่งราคา เพื่อให้ server ตรวจราคาจากฐานข้อมูลเอง -->

<?php require_once __DIR__ . '/../includes/footer.php'; ?>
```

## 9. ปรับ `actions/add_to_cart.php` ให้รองรับ `return_to`

ไฟล์ที่ต้องแก้

```text
actions/add_to_cart.php
```

ของเดิมหลังเพิ่มสินค้าแล้วจะ redirect ไปที่ตะกร้าเสมอ

```php
flash('success', 'เพิ่มลงตะกร้าแล้ว');
redirect('student/cart.php');
```

ควรแก้เป็น

```php
flash('success', 'เพิ่มลงตะกร้าแล้ว');

// ให้ form กำหนดได้ว่าจะกลับไปหน้าไหนหลังเพิ่มสินค้า
// แต่ต้องจำกัดเฉพาะหน้าที่ปลอดภัย เพื่อป้องกัน open redirect
$returnTo = (string) ($_POST['return_to'] ?? 'student/cart.php');

$allowedReturns = [
    'student/cart.php',
    'student/products.php',
    'student/scan.php',
];

if (!in_array($returnTo, $allowedReturns, true)) {
    $returnTo = 'student/cart.php';
}

redirect($returnTo);
```

เหตุผลที่ต้องใช้ whitelist

- ป้องกันไม่ให้ user ส่ง URL ภายนอกเข้ามา
- ป้องกัน open redirect
- ควบคุม flow ของระบบให้ปลอดภัย

## 12. เพิ่มไฟล์ `actions/bulk_add_to_cart.php`

สร้างไฟล์ใหม่

```text
actions/bulk_add_to_cart.php
```

ตัวอย่างโค้ดตั้งต้น

```php
<?php

// เปิด strict mode เพื่อช่วยลดข้อผิดพลาดของชนิดข้อมูล
declare(strict_types=1);

require_once __DIR__ . '/../includes/helpers.php';
require_once __DIR__ . '/../includes/csrf.php';

// ตรวจ CSRF ก่อนทำงานทุกครั้ง
verify_csrf();

// ตรวจว่านักเรียนเลือกกลุ่มแล้วหรือยัง
$group = require_student_group();

// รับจำนวนสินค้าจาก form
$quantities = $_POST['qty'] ?? [];

if (!is_array($quantities)) {
    flash('warning', 'ข้อมูลรายการสินค้าไม่ถูกต้อง');
    redirect('student/products.php');
}

// ดึงหรือสร้างตะกร้าปัจจุบันของกลุ่ม
$cart = get_or_create_open_cart((int) $group['activity_id'], (int) $group['id']);

$addedCount = 0;

foreach ($quantities as $productId => $quantity) {
    $productId = (int) $productId;
    $quantity = max(0, min(20, (int) $quantity));

    // ข้ามสินค้าที่ไม่ได้เลือก
    if ($quantity <= 0) {
        continue;
    }

    // ตรวจว่าสินค้าอยู่ในกิจกรรมเดียวกันและเปิดขายอยู่
    $stmt = db()->prepare(
        "SELECT p.id
         FROM products p
         JOIN activities a ON a.id = p.activity_id
         WHERE p.id = ?
           AND p.activity_id = ?
           AND p.is_active = 1
           AND a.status = 'active'"
    );
    $stmt->execute([$productId, $group['activity_id']]);
    $product = $stmt->fetch();

    if (!$product) {
        continue;
    }

    // เพิ่มลงตะกร้า ถ้ามีอยู่แล้วให้บวกจำนวน
    $stmt = db()->prepare(
        'INSERT INTO cart_items (cart_id, product_id, quantity)
         VALUES (?, ?, ?)
         ON DUPLICATE KEY UPDATE quantity = LEAST(quantity + VALUES(quantity), 99)'
    );
    $stmt->execute([$cart['id'], $productId, $quantity]);

    $addedCount++;
}

if ($addedCount <= 0) {
    flash('warning', 'กรุณาเลือกสินค้าอย่างน้อย 1 รายการ');
    redirect('student/products.php');
}

flash('success', 'เพิ่มสินค้าลงตะกร้าแล้ว');
redirect('student/cart.php');
```

## 13. ปรับ `student/products.php` สำหรับ Phase 2

ใน Phase 2 หน้า `student/products.php` จะเปลี่ยนจาก form ย่อยในแต่ละ card เป็น form ใหญ่ครอบทั้งหน้า

แนวคิด UI

```text
รายการสินค้า

ข้าวสวย       [-] 0 [+]
ไข่ต้ม        [-] 0 [+]
นมจืด         [-] 0 [+]
กล้วย         [-] 0 [+]

[เพิ่มรายการทั้งหมดลงตะกร้า]
```

ข้อดี

- เด็กเลือกหลายอย่างได้เร็ว
- ลดการ redirect หลายครั้ง
- เหมาะกับ tablet หรือมือถือกลุ่มเดียว
- ครูควบคุมกิจกรรมง่ายขึ้น

ข้อควรระวัง

- ค่าเริ่มต้นควรเป็น `0` ไม่ใช่ `1` เพื่อไม่เผลอเพิ่มสินค้าทุกอย่าง
- ต้องตรวจฝั่ง server ทุกครั้งเหมือนเดิม
- ยังคงไม่ส่งราคาไปจาก form

# 14. ผลกระทบต่อระบบเดิม

การเพิ่ม `student/products.php` จะไม่กระทบระบบเดิมมาก เพราะ

- ไม่ต้องแก้ database
- ไม่ต้องลบ QR Code เดิม
- ไม่ต้องแก้ checkout หลัก
- ไม่ต้องแก้รายงานครู
- ไม่ต้องแก้ `orders` และ `order_items`
- ไม่ต้องแก้ `wallet_transactions`

ระบบที่มีอยู่แล้วจะยังทำงานเหมือนเดิม

```text
QR Code → product.php → add_to_cart.php → cart.php → checkout.php
```

เพิ่มเพียง flow ใหม่

```text
products.php → add_to_cart.php หรือ bulk_add_to_cart.php → cart.php → checkout.php
```

# 15. ความเสี่ยงและแนวทางป้องกัน

## 15.1 ความเสี่ยง: นักเรียนเห็นสินค้าทั้งหมดแล้วเดาได้ง่ายขึ้น

แนวทางป้องกัน

- ห้ามแสดงหมู่อาหาร
- ห้ามแสดงสีแยกหมวด
- ห้ามแสดง icon หมวดอาหาร
- ใช้รายละเอียดสินค้าแบบเป็นกลาง
- ให้เด็กอภิปรายหลังจบกิจกรรมแทนการเฉลยระหว่างเล่น

---

## 15.2 ความเสี่ยง: นักเรียนกดเพิ่มสินค้าซ้ำเยอะเกินไป

ระบบเดิมจำกัดจำนวนใน `actions/add_to_cart.php` ไว้ที่ 1–20 ต่อครั้ง และในตะกร้ารวมไม่เกิน 99 จาก `LEAST(quantity + VALUES(quantity), 99)`

แนวทางป้องกันเพิ่มเติม

- UI ควรแสดงตะกร้าให้ชัดเจน
- หน้า products ควรมีปุ่มไปตะกร้าเด่น
- ถ้าจำเป็น ให้ลด max จาก 20 เป็น 10 สำหรับเด็กเล็ก

---

## 15.3 ความเสี่ยง: redirect ไม่ปลอดภัย

ถ้าเพิ่ม `return_to` ต้องใช้ whitelist เสมอ ห้าม redirect ไปค่าที่ user ส่งมาโดยตรง

ตัวอย่าง whitelist

```php
$allowedReturns = [
    'student/cart.php',
    'student/products.php',
    'student/scan.php',
];
```

---

## 15.4 ความเสี่ยง: สินค้าปิดขายแล้วยังอยู่ในหน้าเก่า

แนวทางป้องกัน

- หน้า `student/products.php` ต้องดึงเฉพาะ `is_active = 1`
- `actions/add_to_cart.php` ต้องตรวจซ้ำอีกครั้ง ซึ่งระบบเดิมทำอยู่แล้ว
- `actions/checkout.php` ต้องตรวจซ้ำอีกครั้ง ซึ่งระบบเดิมทำอยู่แล้ว
